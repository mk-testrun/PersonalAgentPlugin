description: Nutze wenn Stack, Build-Prozess, Tests und Verzeichnisstruktur eines Repos erklärt werden sollen.
---

## Schritte

1. `package.json` / `*.sln` / `*.csproj` finden → Stack ableiten
2. Build-Befehl aus README/scripts ermitteln
3. Test-Befehl und Framework ableiten
4. Wichtigste Verzeichnisse beschreiben (max. 3 Ebenen)
5. `.gitignore` auf Besonderheiten prüfen
